# 📅 更新策略说明

## ⏰ 问题1：是否每天6点更新？

### ✅ 是的！每天北京时间早上 6:00 自动更新

**技术实现：**
- 使用 Netlify Scheduled Functions（定时函数）
- 配置文件：`netlify.toml`
- Cron 表达式：`0 22 * * *`（UTC 时区）
- 时区转换：UTC 22:00 = 北京时间 06:00（第二天早上）

**注意事项：**
- ⚠️ Netlify 使用 **UTC 时区**，不是北京时间
- ⚠️ 定时任务只在**生产环境**（published deploys）执行
- ⚠️ 部署预览（Deploy Previews）和分支部署不会触发定时任务
- ✅ 可以在 Netlify 控制台的 Functions 页面点击 "Run now" 手动测试

#### 第2步：数据清洗
- 跳过无效条目（无标题、标题为 "No title"）
- 解析 RSS 2.0 和 Atom 格式
- 提取字段：标题、摘要、发布日期、原文链接

---

#### 第3步：去重
- 根据 **URL** 去重（避免不同源报道同一新闻）
- 根据 **标题** 去重（防止标题相同但URL不同的情况）
- 去重算法：
  ```javascript
  const seenUrls = new Set();
  const seenTitles = new Set();
  
  for (const news of allNews) {
    const urlKey = news.url.toLowerCase().trim();
    const titleKey = news.title.toLowerCase().trim();
    
    if (!seenUrls.has(urlKey) && !seenTitles.has(titleKey)) {
      // 保留这条新闻
    }
  }
  ```

---

#### 第4步：排序
- 按 **发布日期** 降序排序（最新的在前）
- 日期解析：支持 RFC 822 格式（`Mon, 29 Jun 2026 08:00:00 GMT`）

---

#### 第5步：返回
- 返回 **前 20 条**（保证质量，避免加载过慢）
- 如果某个源失效，会自动添加**备用数据**（3条示例新闻）

## 🔧 如何修改更新策略？

### 修改每源条数
编辑 `netlify/functions/fetch-news.js`，修改 `maxItems` 字段：

```javascript
const INTERNATIONAL_SOURCES = [
  {
    name: 'Science',
    maxItems: 5  // 改成 5 条
  },
  // ...
];
```

### 修改更新时间
编辑 `netlify.toml`，修改 cron 表达式：

```toml
[functions."fetch-news"]
  schedule = "0 22 * * *"  # UTC 22:00 = 北京时间 06:00
```

**常用 cron 表达式：**
- `0 22 * * *` - 每天北京时间 06:00
- `0 14 * * *` - 每天北京时间 22:00
- `0 22 * * 1-5` - 工作日北京时间 06:00
- `@daily` - 每天 UTC 00:00（北京时间 08:00）

### 添加更多 RSS 源
编辑 `netlify/functions/fetch-news.js`，在 `INTERNATIONAL_SOURCES` 或 `DOMESTIC_SOURCES` 数组中添加新源：

```javascript
const INTERNATIONAL_SOURCES = [
  // ... 现有源
  {
    name: 'New Journal Name',
    nameCn: '新期刊名称',
    url: 'https://www.example.com/rss',
    verified: false,  // 需要先验证 RSS 源是否可用
    maxItems: 3
  }
];
```

- ⚠️ 期刊可能会更改 RSS URL
- ⚠️ 如果某个源失效，定时任务会继续执行，但会跳过失效的源
- 📋 建议定期检查 Netlify Functions 日志

### 关于 Netlify 定时任务的限制
- ⏱️ 执行时间限制：30 秒（Background Functions 可延长）
- 💰 免费额度：125,000 次调用/月（足够每日定时任务使用）
- 🚀 如果需要更长执行时间，可以改用 Background Functions
