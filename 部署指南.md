# 🌐 BioJournal 网站 - 免费部署指南

## 📋 免费托管平台对比

| 平台 | 免费域名 | 优势 | 限制 |
|------|----------|------|------|
| **GitHub Pages** | `username.github.io/repo` | 最稳定、SEO友好 | 需要GitHub账号 |
| **Vercel** | `project.vercel.app` | 速度快、自动HTTPS | 需要GitHub账号 |
| **Netlify** | `project.netlify.app` | 简单易用、表单支持 | 需要账号 |
| **Cloudflare Pages** | `project.pages.dev` | 全球CDN、无限带宽 | 需要Cloudflare账号 |

**推荐方案：GitHub Pages**（最稳定，适合长期运营）

---

## 🚀 方案一：部署到 GitHub Pages（推荐）

### 第一步：创建 GitHub 账号
1. 访问 [github.com](https://github.com/)
2. 点击 "Sign up" 注册账号
3. 验证邮箱

### 第二步：创建仓库
1. 登录后点击右上角 `+` → `New repository`
2. 仓库名填写：`biojournal-website`
3. 选择 `Public`
4. 勾选 `Add a README file`
5. 点击 `Create repository`

### 第三步：上传文件
**方法A：网页上传（最简单）**
1. 进入刚创建的仓库
2. 点击 `Add file` → `Upload files`
3. 将 `biojournal-website` 文件夹内的所有文件拖拽上传
4. Commit message 填写：`Initial commit`
5. 点击 `Commit changes`

**方法B：Git 命令上传（推荐）**
```bash
# 在你的电脑上执行
cd /Users/mac/Workbuddy/2026-06-28-23-11-02/biojournal-website
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/你的用户名/biojournal-website.git
git push -u origin main
```

### 第四步：启用 GitHub Pages
1. 进入仓库页面
2. 点击 `Settings` 标签
3. 左侧菜单找到 `Pages`
4. `Source` 选择 `Deploy from a branch`
5. `Branch` 选择 `main` 和 `/ (root)`
6. 点击 `Save`
7. 等待 1-2 分钟

### 第五步：访问你的网站！
免费域名格式：
```
https://你的用户名.github.io/biojournal-website/
```

**示例**：如果用户名是 `zhangshan`，则网站地址是：
```
https://zhangshan.github.io/biojournal-website/
```

## 🚀 方案二：一键部署到 Vercel（最快）

### 第一步：准备 GitHub 仓库
（同上，先上传代码到 GitHub）

### 第二步：Vercel 部署
1. 访问 [vercel.com](https://vercel.com/)
2. 点击 `Sign Up`，选择 `Continue with GitHub`
3. 授权后点击 `New Project`
4. 选择 `biojournal-website` 仓库
5. 点击 `Deploy`
6. 等待 1 分钟...

### 第三步：获得免费域名
Vercel 会自动分配一个域名：
```
https://biojournal-website.vercel.app
```

**的优势**：
- ✅ 自动 HTTPS
- ✅ 全球 CDN 加速
- ✅ 每次 git push 自动更新

## 🚀 方案三：部署到 Netlify（最简单）

### 方法A：拖拽上传（无需GitHub）
1. 访问 [netlify.com](https://netlify.com/)
2. 注册账号并登录
3. 进入 `Sites` 页面
4. **直接拖拽** `biojournal-website` 文件夹到页面中央
5. 等待上传...

### 获得免费域名：
```
https://随机名称.netlify.app
```

**自定义域名**：在 `Domain settings` 中可以修改子域名

## 📞 需要帮助？

如果遇到问题，可以：
1. **GitHub Pages 无法访问**：检查仓库是否为 Public，检查 `Settings` → `Pages` 配置
2. **样式丢失**：检查 `index.html` 中的 CSS/JS 路径是否正确
3. **Vercel 部署失败**：检查 `index.html` 是否在根目录
